# feat: V1.1 Enhancements

## Overview

Six enhancements that move Calendar Click from a functional V1 to a daily-driver tool: global keyboard shortcut, smart slot rounding, exposed minimum-slot setting, preview popover, recipient timezone conversion, and Markdown output format.

## Problem Frame

V1 works but has friction for power users: mouse-only activation, machine-precise times (10:47am), a hidden setting, no way to verify output before pasting, no cross-timezone support, and single-format output. These six fixes address the highest-leverage pain points identified in ideation. (see origin: docs/brainstorms/2026-03-26-v1.1-enhancements-requirements.md)

## Requirements Trace

- R1. Global keyboard shortcut (Ctrl+Shift+C default, configurable, clearable)
- R2. Smart slot rounding (configurable 5/10/15/30 min, default 30, "off" option)
- R3. Expose minimum slot duration in Settings (15/30/45/60 min picker)
- R4. Preview popover on Option+click (shows text, Copy button, dismisses on click-outside)
- R5. Recipient timezone conversion (picker in popover, search, recent timezones saved)
- R6. Format templates (Plain Text default + Markdown, selectable in Settings and popover)

## Scope Boundaries

- No HTML rich text format (deferred to V1.2)
- No drag-and-drop from icon
- No auto-paste or scheduling-intent detection
- No availability diffing
- Keyboard shortcut triggers default copy only — no shortcut for preview
- Left-click never affected by popover timezone/format selections

## Context & Research

### Relevant Code and Patterns

- `AppDelegate.swift` — owns the copy pipeline: `copyDefault()` → `copyRange()` → fetch → calculate → format → clipboard → feedback
- `StatusItemController.swift` — click handling via `NSApp.currentEvent.type` and `event.modifierFlags`; Option+click distinguished with `.contains(.option)`
- `AvailabilityService.swift` — `subtractEvents()` produces exact-boundary slots; rounding fits after subtraction, before min-duration filter
- `AvailabilityFormatter.swift` — pure function `format(slots:showTimeZone:)` → `String`; uses `Calendar.current.component(.hour/.minute)` for time extraction
- `AppSettings.swift` — enum pattern: define key, default, register, clamped getter
- `SettingsView.swift` — `SettingsSection` component, `@AppStorage` bindings, `ScrollView` > `VStack`
- `CalendarClickTests.swift` — Apple Testing framework, `@Suite`/`@Test`, helper functions `date()` and `slot()`

### Key Technical Decisions Resolved

**Q1 (R1): Global hotkey API** — Use `NSEvent.addGlobalMonitorForEvents(matching: .keyDown)` + `NSEvent.addLocalMonitorForEvents(matching: .keyDown)` to cover background and foreground. No Accessibility permission required for read-only monitoring. For the shortcut recorder in Settings, use a custom SwiftUI view that wraps a first-responder `NSView` to capture key combos. Store as a dictionary `{keyCode: Int, modifiers: Int}` in UserDefaults.

**Q2 (R4): Popover type** — Use `NSPopover` attached to the status item button. Standard macOS pattern for menu bar popovers. Auto-dismisses on click-outside, handles anchoring, hosts SwiftUI via `NSHostingController`.

**Q3 (R5): Timezone search** — Use `TimeZone.knownTimeZoneIdentifiers` (~400 entries) with `NSTimeZone.localizedName()` for display. Filter on both identifier ("America/New_York") and display name ("Eastern Time"). No bundled dataset needed.

**Q4 (R2): Rounding location** — Round in `AvailabilityService` after `subtractEvents()` returns, before the min-duration filter. This keeps the formatter pure and ensures rounding applies regardless of output format.

- [ ] **Unit 2: Smart slot rounding in AvailabilityService**

  **Goal:** Round free-slot boundaries to configurable increments after subtraction.

  **Requirements:** R2

  **Dependencies:** Unit 1

  **Files:**
  - Modify: `CalendarClick/Services/AvailabilityService.swift`
  - Test: `CalendarClickTests/CalendarClickTests.swift`

  **Approach:**
  - Add `roundUp(_ date: Date, toMinutes: Int) -> Date` and `roundDown(_ date: Date, toMinutes: Int) -> Date` helpers (using `Calendar.date(byAdding:)` — not raw arithmetic)
  - In `calculateAvailability()`, after `subtractEvents()` returns and before min-duration filter: round each slot's start up and end down
  - If rounding makes start >= end, drop the slot
  - If granularity is 0 (off), skip rounding

  **Patterns to follow:** Existing `dateFromMinutes()` helper style; `Calendar.date(byAdding:)` for safe date math

  **Test scenarios:**
  - 10:47am rounds up to 11:00am (30-min granularity)
  - 2:08pm rounds down to 2:00pm (30-min granularity)
  - 9:00am stays 9:00am (already on boundary)
  - 10:47am rounds to 10:50am (10-min granularity)
  - Slot becomes negative after rounding → dropped
  - Slot becomes shorter than minimum after rounding → dropped
  - Granularity 0 → no rounding applied
  - Rounding preserves date (doesn't cross midnight)

  **Verification:** New rounding tests pass; existing slot subtraction tests unaffected

## System-Wide Impact

- **Interaction graph:** Option+click adds a new branch in `StatusItemController.handleClick()`. Global shortcut adds a parallel entry point via `GlobalShortcutManager` → `AppDelegate.copyDefault()`. Both converge on the existing copy pipeline.
- **Error propagation:** No new error paths — shortcut and popover both use existing `copyRange()` pipeline. Permission denied is handled the same way.
- **State lifecycle risks:** Shortcut storage is a simple dictionary in UserDefaults. Recent timezones is a capped array. No persistence complexity.
- **API surface parity:** `AvailabilityFormatter.format()` gains new parameters but defaults preserve backward compatibility. All existing callers work unchanged.
- **Integration coverage:** Rounding + minimum-slot interaction needs testing (rounding can shrink slots below minimum). Timezone conversion + suffix elision rules need testing.

## Risks & Dependencies

| Risk | Mitigation |
|------|------------|
| Global shortcut conflicts with other apps | Use uncommon combo (Ctrl+Shift+C); make it configurable and clearable |
| `addGlobalMonitorForEvents` may not fire in all contexts (e.g., secure input fields) | Document limitation; this matches what Rectangle/Raycast users expect |
| Timezone conversion edge case: slot crosses midnight in recipient's zone | Test explicitly; the formatter already handles cross-day scenarios |
| Rounding + minimum-slot interaction drops too many slots | Test that rounding respects minimum; warn in Settings if aggressive rounding + high minimum |
| NSPopover focus issues with menu bar apps | Test focus/dismiss behavior; fall back to NSPanel if popover misbehaves |
